# MiniWorldSprites 素材库使用说明书

## 1. 简介

本素材库 (`MiniWorldSprites`) 是一个像素风格的2D游戏资源包，适用于俯视角（Top-Down）的策略、RPG或模拟类游戏。其风格统一，资源丰富，涵盖了角色、建筑、自然环境、UI等多种元素。

**核心特性:**
- **风格:** 统一的像素画风 (Pixel Art)。
- **尺寸:** 基础网格尺寸为 `16x16` 或 `32x32` 像素，大部分资源都遵循这个规范。
- **组织:** 文件夹分类清晰，便于按类别查找。
- **命名:** 文件名包含丰富信息，如尺寸、颜色、类型等。

---

## 2. 目录结构概览

素材库的根目录按照资源类型进行了清晰的划分：

- **/Animals**: 包含马、海洋生物等动物精灵。
- **/Buildings**: 游戏中的所有建筑结构。
- **/Characters**: 核心的游戏角色，包括玩家可控单位、怪物和NPC。
- **/Ground**: 用于铺设地图的地面纹理，如草地、沙地、悬崖。
- **/Miscellaneous**: 无法归入其他大类的杂项物品，如桥梁、船只、传送门。
- **/Nature**: 构成自然环境的元素，如树木、岩石、仙人掌。
- **/Objects**: 可被角色使用或交互的小物件，主要是武器和投射物。
- **/Templates**: 用于创建新角色的基础模板，定义了不同尺寸的标准。
- **/User Interface**: 用于构建游戏UI的图标和元素。

---

## 3. 重点目录详解与使用建议

### 3.1 `Characters` (角色)
这是最重要的目录之一，包含了所有动态单位。

- **/Champions**: 精英或英雄单位。
  - **用途:** 非常适合用作《深渊低语》中的**“守望者”**或特殊NPC，例如 `Arthax.png` 或 `Katan.png` 可以作为圣骑士或退役老兵的形象。
- **/Monsters**: 敌人单位。
  - **子目录:** 进一步按种族划分（恶魔、龙、兽人、亡灵等）。
  - **用途:** 可用作玩家在游戏后期召唤的**魔物**，例如 `Demons/RedDemon.png` 或 `Undead/Skeleton-Soldier.png`。
- **/Soldiers**: 人类士兵单位。
  - **子目录:** 按兵种划分（近战、远程、骑兵）。
  - **用途:** 适合作为“圣橡镇”的**城镇卫兵**。
- **/Workers**: 工人或平民单位。
  - **用途:** 适合作为游戏中的普通**NPC**，如农夫、铁匠、商人等。`FarmerTemplate.png` 是一个很好的基础。

### 3.2 `Buildings` (建筑)
用于构建“圣橡镇”的场景。

- **/Wood**: 通用的木质结构建筑。
  - **用途:** 这是构建基础小镇的核心，包含 `Houses.png`, `Taverns.png`, `Market.png` 等。
- **/Colored (Cyan, Lime, Purple, Red)**: 按颜色划分的派系建筑。
  - **用途:** 在游戏中可以代表被不同势力（或被玩家“腐化”）的区域。例如，一个区域的建筑可以从 `Wood` 变为 `Purple` 来表示该地已被侵蚀。
- **/Enemy**: 敌方建筑。
  - **用途:** 可用于兽人营地或被彻底腐化的区域。

### 3.3 `Ground` & `Nature` (环境)
这两个目录共同构建了游戏世界的背景。

- `Ground/Grass.png`, `Ground/TexturedGrass.png`: 构成圣橡镇及周边的基础地形。
- `Ground/Cliff.png`, `Ground/Shore.png`: 用于制作河流、湖泊或海岸的边界。
- `Nature/Trees.png`, `Nature/Rocks.png`: 填充场景，使其更加生动。可以作为玩家初期进行“微操”的对象（如：让石头滚落）。

### 3.4 `Objects` (物品)
包含了游戏中可交互或作为动画效果的小物件。

- **用途:**
  - `ArrowLong.png`, `FireballProjectile.png`: 可用于战斗动画或作为玩家能力的视觉效果。
  - `Axe.png`, `SwordShort.png`: 可作为NPC携带的道具，或在特定场景（如铁匠铺）作为环境装饰。

### 3.5 `User Interface` (UI)
- `BoxSelector.png`: 游戏中的单位选择框。
- `Highlighted-Boxes.png`: 高亮或目标指示框。
- `UiIcons.png`: 可能是一个包含多种UI图标的**精灵图表 (Sprite Sheet)**，需要分割使用。

## 4. 命名约定与使用技巧

- **尺寸后缀 `(32x32)`:** 文件名中的尺寸信息（如 `Horse(32x32).png`）明确了该资源的基础尺寸，便于开发者在引擎中设置碰撞盒或对齐网格。
- **颜色前缀 `Cyan...`:** 在 `Buildings` 目录中，颜色前缀用于区分不同派系的建筑。
- **精灵图表 (Sprite Sheets):** 一些文件名（如 `MarineAnimals.png`）可能暗示这是一个包含多帧动画或多个角色的精灵图表。使用时，需要在游戏引擎中将其切割为独立的帧或单位。请参考下面的实例。
- **模板 `Templates`:** `16x16Small.png` 或 `32x32Small.png` 提供了角色创建的标准尺寸，进行自定义绘制时应以此为基准，以保证游戏整体风格的统一。

### 4.1 精灵图表分析实例: `Animals/MarineAnimals.png`

精灵图表将多个帧或多个图像合并在一张图上以提高性能。要使用它，我们必须知道其中每个元素的坐标和尺寸。以上一节分析过的 `Animals/MarineAnimals.png` 为例：

**分析:**

1.  **布局:** 这是一个典型的网格布局图。通过观察，我们可以推断出单个精灵（帧）的基础尺寸很可能是 `32x32` 像素。
2.  **内容:**
    *   **左侧 (前2列):** 一只海龟，拥有4个方向（4行），每个方向有2帧动画（2列）。
    *   **右侧 (第3列起):** 多种不同颜色的人形/鱼形生物，每个生物的动画帧水平排列。

**在游戏引擎中的切割方法:**

假设坐标系原点 `(0, 0)` 在左上角。

-   **海龟“向下走”动画:**
    -   第1帧: `(x: 0, y: 0, width: 32, height: 32)`
    -   第2帧: `(x: 32, y: 0, width: 32, height: 32)`
-   **海龟“向左走”动画 (第二行):**
    -   第1帧: `(x: 0, y: 32, width: 32, height: 32)`
    -   第2帧: `(x: 32, y: 32, width: 32, height: 32)`
-   **第一个青色生物动画:**
    -   第1帧: `(x: 64, y: 0, width: 32, height: 32)`
    -   第2帧: `(x: 96, y: 0, width: 32, height: 32)`

**开发者总结:**
在代码中，你需要定义好这些矩形区域（Rectangles/Textures），然后根据需要渲染或播放对应的动画序列。为了方便管理，强烈建议将这些信息配置化（例如，存储在一个或多个JSON文件中），而不是在代码里硬编码。

## 5. 结语

`MiniWorldSprites` 是一个结构优秀、内容丰富的素材库。通过本说明书，开发者可以快速定位所需资源，并理解其设计意图，从而高效地将其应用于《深渊低语：封印纪元》的游戏开发中。建议在使用前，花少量时间在图像编辑软件中查看那些可能是精灵图表的文件，以确认其内部布局。
